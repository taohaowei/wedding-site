## 二、对接契约(三方共同遵守)

### 2.1 前端 ↔ 后端:HTTP API

| 端点 | 方法 | 请求 | 响应 |
|---|---|---|---|
| `/api/health` | GET | - | `{ok:true,ts:number}` |
| `/api/rsvp` | POST | `{name,attending:'yes'|'no'|'maybe',headcount?:number,need_lodging?:'yes'|'no'|'self',dietary?:string,message?:string}` | `{ok:true,id:number}` 或 `{ok:false,error:string}` |
| `/api/admin/login` | POST | `{password:string}` | `{ok:true}` + Set-Cookie `admin_session=token; HttpOnly; Secure; SameSite=Lax; Path=/` |
| `/api/admin/logout` | POST | (cookie) | `{ok:true}` |
| `/api/admin/rsvps` | GET | (cookie) | `{ok:true,data:RSVPRecord[]}` |
| `/api/admin/export.csv` | GET | (cookie) | `text/csv; charset=utf-8` 文件下载 |

**校验规则**(双方共识):
- `name`:1-32 字符
- `attending`:必填枚举
- `headcount`:1-20 整数,仅当 `attending='yes'`
- `dietary`/`message`:0-200 字符,可选
- 后端用 zod 校验,失败返回 400 + `{ok:false,error:'描述'}`

### 2.2 前端 ↔ Nginx:产物结构

```
dist/
├── index.html
├── assets/
│   ├── *.js
│   ├── *.css
│   └── *.{png,jpg,webp,svg,mp3}
└── public 资源同名
```
- 路由:**SPA fallback**,Nginx 必须配 `try_files $uri /index.html`
- 静态资源带 hash,可长缓存

### 2.3 后端 ↔ Nginx:监听端口

- Hono 监听 `127.0.0.1:3001`(只本地)
- Nginx 反代 `/api/` → `http://127.0.0.1:3001/api/`
- DB 文件:`/var/lib/wedding/wedding.db`(由 PM2 启动用户读写)

## 三、TDD 行为断言(必须通过才算交付)

### 3.1 RSVP 提交

| # | 给定 | 当 | 预期 |
|---|---|---|---|
| 1 | 合法表单 | POST /api/rsvp | 200 + 记录入库 + 返回 id |

## 六、关键风险

1. **微信 X5 内核**:Swiper 嵌套手势在某些 X5 版本可能丢手势 → 必须真机测试
2. **iOS 100vh**:统一 `100dvh` + 旧版用 JS 兜底
3. **音乐 autoplay**:微信内必须用 `WeixinJSBridgeReady`,默认右上角小喇叭按钮静默
4. **真人参考照插画**:Q 版相似度有上限(并行不阻塞,稍后处理)
5. **部署密钥**:`ADMIN_PASSWORD` 必须用环境变量,**不能进 git**
